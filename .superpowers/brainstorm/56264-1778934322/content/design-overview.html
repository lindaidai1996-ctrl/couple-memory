P1-9：骨架屏 + 预加载

覆盖范围 & 技术方案
动画风格
Shimmer（微光扫过）— 左→右高光横扫，后台用 warm-border/warm-sidebar 色，公开页用 film-surface 色
覆盖页面
后台 (warm-*)：照片网格、相册列表、设置页表单、照片详情 Modal、Sidebar 导航
公开页 (film-*)：PhotoStream、Timeline、首页 Hero
预加载策略
IntersectionObserver + rootMargin: "200px" — 图片进入视口前 200px 触发加载，配合骨架屏实现无缝过渡
组件设计
• 通用 <Skeleton> 基础组件（支持 width/height/radius/variant 参数）
• 场景组合组件：<PhotoGridSkeleton>、<TimelineSkeleton>、<SettingsFormSkeleton> 等
• CSS keyframes 定义在 globals.css，通过 Tailwind 自定义动画类 animate-shimmer 使用
• 后台用纯 CSS（遵循"后台纯 CSS transition"原则），公开页骨架屏配合 Framer Motion 入场

P1-10：错误边界

架构 & UI 设计
粒度
页面级 Error Boundary — 每个路由页面顶层包裹，崩溃时整页替换为错误 UI
视觉风格
大字排版 — 巨型装饰文字 "Oops" 作为视觉焦点 + 友好文案 + 双操作按钮（刷新/返回首页）
主题适配
• 后台页面：warm-* 配色，"Oops" 用 warm-border 浅色，按钮用 warm-accent
• 公开页面：film-* 配色，"Oops" 用 film-surface 深灰色，按钮用 film-accent
技术实现
• Next.js App Router 内置 error.tsx 约定（每个路由段可放置）
• 根 app/error.tsx 作为全局兜底
• app/global-error.tsx 捕获 layout 级错误（含根 layout）
• 提供 reset() 刷新功能 + router.push('/') 返回首页

P1-11：邀请链接页面

双端设计
Partner 接受页（film-* 暗色）
• 全屏暗色背景 + film-grain 纹理
• 居中浮层卡片（film-surface 底 + accent 边框微光）
• 双头像（Owner 头像 + "?" 占位暗示配对关系）
• 空间名称 + 邀请文案 + "加入空间"按钮
• 底部小字过期倒计时
• 异常态：过期（410）、已满员（409）、已是成员（409）各有对应 UI
Owner 邀请管理（warm-* 亮色，Settings 页内）
• 嵌入现有 Settings 页面作为独立 section
• 显示当前邀请码 + 完整链接 + 一键复制
• 有效期倒计时展示
• "重新生成"按钮（旧码立即失效）
• 无邀请码时展示"生成邀请链接"入口
路由结构
• 接受页：/invite/[code]（独立路由，不走 dashboard layout）
• 管理 UI：/(dashboard)/settings 页面内新增 section
