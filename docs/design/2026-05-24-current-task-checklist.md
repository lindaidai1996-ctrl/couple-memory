## 2. 公开页叙事层体验打磨

### 目标

在不扩写新功能的前提下，提升公开页从 `Hero` 进入叙事层时的阅读节奏和表达完整性。

### 核心任务

- [x] 检查相册卡之间的节奏感是否自然
- [x] 检查章节摘要篇幅是否过长、过短或密度不均
- [x] 检查 `Hero` 与 `Narrative` 的视觉衔接是否顺畅
- [x] 归纳当前公开页叙事层的主要体验问题
- [x] 明确哪些问题属于本轮应修，哪些问题应延后

### 完成标准

- [x] 可以列出一份明确的公开页叙事层体验问题列表
- [x] 每个问题都有处理优先级，而不是继续泛化讨论

### 2026-05-24 打磨结果

- 已处理问题：
  - 相册卡连续同构导致首页叙事节奏过平。
  - 相册说明与章节摘要在首页叙事层过长时会挤压阅读节奏。
  - `Hero -> Narrative` 的切换缺少明确的视觉缓冲层。
- 已落地调整：
  - 公开页叙事卡增加左右交替布局节奏。
  - 相册说明在首页叙事层改为较短摘录展示。
  - 章节摘要在首页叙事层改为较短摘录展示。
  - 有封面图时，`Narrative` 段新增抬升、圆角和渐变承接，弱化整屏 Hero 与正文段落的硬切。
- 当前优先级判断：
  - `P0`: 已处理首页叙事层节奏、篇幅、衔接问题。
  - `P1`: 如果后续继续打磨，应优先观察真实公开数据下的多相册排布效果。
  - `P2`: 章节摘要的更细粒度文风优化可以延后，不阻塞当前窗口。
- 代码与测试证据：
  - 代码：`src/app/s/[slug]/page.tsx`
  - 测试：`tests/app/public-home-page.test.ts`

### 完成标准

- [x] 能解释为什么当前不并行推进其他 `P2` 候选项

### 2026-05-24 决策结论

- 已确认 `P2` 首个启动方向为：`年度回顾 / 周年回顾`
- 依据：
  - `docs/design/2026-05-22-p1-closure-summary.md` 已把这一路径列为下一窗口优先启动候选之一。
  - `docs/superpowers/specs/2026-05-22-p2-memory-review-design.md` 已将 `P2` 首批能力收敛为回顾页闭环。
  - `docs/superpowers/plans/2026-05-22-p2-memory-review-phase1.md` 已把这一切口展开为可执行实现计划。
- 当前不并行推进其他 `P2` 候选项的原因：
  - `风格长期记忆` 更偏底层长期模型收敛，不如回顾页直观形成“记忆资产”感知。
  - `“我们的第一次”专题页` 适合作为下一批专题层能力，而不是 `P2` 第一刀。
  - 回顾页已具备现成 spec、plan、页面、路由和测试，验证成本最低、完整度最高。
