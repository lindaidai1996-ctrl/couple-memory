## P0：让 AI 可协作

### 目标

让 AI 从黑箱处理器变成可协作助手，并确认产品主线已经从“单图处理”切换到“章节整理协作”。

### 核心任务

- [x] 核对相册主流程是否已经变成“章节优先”
- [x] 核对“章节”与“其他瞬间”的分工是否清楚
- [x] 核对用户是否能先组织章节，再进入 AI 协作
- [x] 核对单图 AI 是否已经退到辅助层，而不是继续充当主入口
- [x] 核对未归类照片的单图协助是否保持轻量
- [x] 核对已归类照片的单图协助是否体现章节上下文
- [x] 核对 Dashboard 的整理准备度是否在推动用户继续整理章节
- [x] 核对 Dashboard 提示项是否都能落到明确修复动作
- [x] 核对空间级 AI 偏好是否真实影响协作结果感知
- [x] 整理一份 P0 收口结论

### 交付物

- [x] P0 已完成能力清单
- [x] P0 仍需校正的体验问题列表
- [x] P0 是否可以正式收口的结论

### 完成标准

- [x] 用户先整理章节，再使用 AI 协助表达
- [x] 单图 AI 不再抢主流程
- [x] 用户可以感知到 AI 不是一次性输出，而是可协作对象

### 当前核对结果

- 相册页已按“章节在上、其他瞬间在下”的结构组织内容。
- 单图弹窗已根据是否归属章节切换协助文案，符合“章节内二级微调 / 未归类轻协助”的定位。
- Dashboard 已能给出整理准备度分数、提示和直接跳转到相册整理入口的修复动作。
- 设置页中的空间级风格、语气和禁用表达配置已接入照片 AI 文案 prompt，上游生成链路会收到这些偏好。

### P0 收口结论

- P0 主线已经具备收口条件，可以正式进入 P1。
- 当前仍存在的主要差距不再属于 P0 主线阻塞，而是 P1 叙事层缺口：
  - 更完整的里程碑候选表达还未完成。
  - 时间线候选去重与合并建议还未开始。

## P1：让相册有叙事

### 目标

让相册从“照片集合”升级为“可阅读、可确认、可展示的故事单元”。

### 当前已起步项

- [x] 相册叙事进度入口
- [x] 相册标题可编辑
- [x] 相册说明可编辑
- [x] 基于章节生成相册标题草稿
- [x] 基于章节生成相册说明草稿
- [x] AI 里程碑候选支持单个确认、批量确认与待确认提示

### 当前核对结果

- 相册后台页已存在相册叙事卡和标题/说明草稿生成入口。
- 时间线后台页已存在 AI 候选待确认区块，支持关联照片展示、确认、忽略、编辑和批量确认。
- 时间线后台页已补充候选推荐理由展示，并能提示同地点近日期节点的潜在重复关系。
- 时间线后台页的合并建议已提供快速清理重复节点的动作入口。
- 公开页首页已新增相册叙事层，可展示相册标题、相册说明和章节摘要预览。
- 相册后台页已新增封面候选卡片，支持查看当前封面和直接切换封面。
- 相册后台页的叙事卡已补充 AI 草稿与当前版本的对照展示，协作状态更清楚。

### 核心任务

- [x] 完善相册叙事卡
- [x] 明确展示 AI 标题与用户标题
- [x] 明确展示 AI 说明草稿与用户说明
- [x] 提供基于章节重新生成标题的入口
- [x] 提供基于章节重新生成说明的入口
- [x] 实现相册封面候选推荐
- [x] 提供封面候选切换能力
- [x] 提供封面确认能力
- [x] 完善里程碑候选区块
- [x] 展示候选标题
- [x] 展示关联照片
- [x] 展示推荐理由或置信度
- [x] 提供确认、忽略、编辑闭环
- [x] 增加时间线候选去重能力
- [x] 增加时间线候选合并建议
- [x] 在公开页接入相册标题
- [x] 在公开页接入相册说明
- [x] 在公开页接入章节摘要
- [x] 优化公开页阅读顺序，让相册叙事层更连贯
- [x] 清理概念命名，只保留“相册说明”作为唯一相册级叙事文本

### 交付物

- [x] 相册详情页完整叙事卡
- [x] 相册封面候选选择机制
- [x] 里程碑候选确认与维护机制
- [x] 公开页相册叙事展示层

### 完成标准

- [x] 相册具备清晰可读的标题和说明
- [ ] 时间线不再只是按拍摄日期堆叠
- [x] 公开页开始像故事展示页，而不是图片展示页
