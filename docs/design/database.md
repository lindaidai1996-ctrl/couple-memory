# 数据库设计

> 本文档定义数据库表结构、字段规范、索引策略和 Prisma Schema。
>
> 相关文档：[总览](./README.md) · [技术架构](./architecture.md) · [API 设计](./api-design.md)

## ER 关系图

```
User 1──N CoupleUser N──1 Couple
                              │
                    ┌─────────┼─────────┐
                    │         │         │
                  Album    Milestone  PipelineRun
                    │
                  Photo
```

- 一个 User 可以通过 CoupleUser 属于多个 Couple（MVP 限制为 1 个）
- 一个 Couple 下有多个 Album、Milestone、PipelineRun
- 一个 Album 下有多个 Photo
- Photo 不直接关联 Couple，而是通过 Album 间接关联（查询时 JOIN）

## 表结构详细设计

### User — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 用户 ID |
| email | String | UNIQUE, NOT NULL | 登录邮箱 |
| passwordHash | String | NOT NULL | bcrypt 哈希后的密码 |
| name | String? | | 用户昵称 |
| avatar | String? | | 头像 URL |
| createdAt | DateTime | DEFAULT now() | 注册时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

### Couple — 情侣空间表（租户）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 空间 ID |
| slug | String | UNIQUE, NOT NULL | URL 标识，如 "lin-and-xx" |
| name | String | NOT NULL | 空间名称，如 "Lin & XX" |
| startDate | DateTime? | | 在一起的日期 |
| coverPhotoUrl | String? | | 封面照片 URL |
| bio | String? | | "关于我们"简介 |
| theme | String | DEFAULT "default" | 主题标识 |
| isPublic | Boolean | DEFAULT false | 是否公开访问 |
| plan | String | DEFAULT "free" | 付费计划（预留） |
| createdAt | DateTime | DEFAULT now() | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

### CoupleUser — 用户-空间关联表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 关联 ID |
| userId | String | FK → User.id, NOT NULL | 用户 ID |
| coupleId | String | FK → Couple.id, NOT NULL | 空间 ID |
| role | Enum (OWNER/PARTNER) | NOT NULL | 角色 |
| nickname | String? | | 在空间中的昵称 |
| joinedAt | DateTime | DEFAULT now() | 加入时间 |

**联合唯一约束**：`@@unique([userId, coupleId])` — 同一用户不能重复加入同一空间。

### Album — 相册表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 相册 ID |
| coupleId | String | FK → Couple.id, NOT NULL | 所属空间 |
| title | String | NOT NULL | 相册标题 |
| description | String? | | 相册描述 |
| coverPhotoUrl | String? | | 封面照片 URL |
| sortOrder | Int | DEFAULT 0 | 排序权重 |
| createdAt | DateTime | DEFAULT now() | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

### Photo — 照片核心表

这是最复杂的一张表，包含图片基础信息、EXIF 数据、AI 生成内容和处理状态。

**设计决策：为什么不拆分？** 虽然字段多（20+），但这些数据的生命周期一致（随照片创建/删除），查询时也通常一起使用（展示一张照片需要全部信息）。拆分反而增加 JOIN 开销和代码复杂度。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| **基础信息** | | | |
| id | String (cuid) | PK | 照片 ID |
| albumId | String | FK → Album.id, NOT NULL | 所属相册 |
| originalUrl | String | NOT NULL | OSS 原图 URL |
| thumbnailUrl | String? | | 缩略图 URL (300px) |
| displayUrl | String? | | 展示图 URL (1200px) |
| fileName | String | NOT NULL | 原始文件名 |
| fileSize | Int | NOT NULL | 文件大小 (bytes) |
| width | Int? | | 图片宽度 (px) |
| height | Int? | | 图片高度 (px) |
| sortOrder | Int | DEFAULT 0 | 在相册内排序 |
| **EXIF 信息** | | | |
| takenAt | DateTime? | | 拍摄时间（从 EXIF 提取） |
| latitude | Float? | | GPS 纬度 |
| longitude | Float? | | GPS 经度 |
| locationName | String? | | 逆地理编码后的中文地名 |
| cameraMake | String? | | 相机品牌 |
| cameraModel | String? | | 相机型号 |
| focalLength | String? | | 焦距 |
| aperture | String? | | 光圈 |
| shutterSpeed | String? | | 快门速度 |
| iso | Int? | | ISO |
| **AI 生成内容** | | | |
| aiCaption | String? | | AI 生成的文案 |
| userCaption | String? | | 用户自定义文案（优先展示） |
| aiScene | String? | | AI 识别的场景（如 "海边日落"） |
| aiMood | String? | | AI 识别的情绪（如 "温馨浪漫"） |
| aiComposition | String? | | AI 分析的构图（如 "三分法"） |
| aiColorTone | String? | | AI 分析的色调（如 "暖色调"） |
| aiLayout | String | DEFAULT "side-by-side" | AI 推荐的排版模板 |
| aiKeywords | String[]? | | AI 提取的关键词 |
| **状态** | | | |
| status | Enum | NOT NULL | UPLOADING → PROCESSING → READY / FAILED |
| processingError | String? | | 处理失败时的错误信息 |
| createdAt | DateTime | DEFAULT now() | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

**状态机流转**：

```
UPLOADING ──(OSS上传完成)──→ PROCESSING ──(Pipeline完成)──→ READY
                                  │
                                  └──(Pipeline失败)──→ FAILED
```

- `UPLOADING`：前端正在上传到 OSS
- `PROCESSING`：后端正在处理（压缩/EXIF/AI）
- `READY`：处理完成，可展示
- `FAILED`：处理失败，记录错误信息，用户可重试

### Milestone — 里程碑表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 里程碑 ID |
| coupleId | String | FK → Couple.id, NOT NULL | 所属空间 |
| title | String | NOT NULL | 标题，如 "第一次去厦门" |
| description | String? | | 描述 |
| date | DateTime | NOT NULL | 事件日期 |
| locationName | String? | | 地点名称 |
| photoId | String? | FK → Photo.id | 关联照片（可选） |
| isAutoGenerated | Boolean | DEFAULT false | 是否 AI 自动生成 |
| createdAt | DateTime | DEFAULT now() | 创建时间 |
| updatedAt | DateTime | @updatedAt | 更新时间 |

### PipelineRun — Agent 执行记录表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 执行 ID |
| coupleId | String | FK → Couple.id, NOT NULL | 所属空间 |
| photoId | String | FK → Photo.id, NOT NULL | 关联照片 |
| status | Enum | NOT NULL | RUNNING / COMPLETED / FAILED |
| dag | Json | NOT NULL | DAG 定义（节点+依赖） |
| nodeResults | Json? | | 各节点执行结果 |
| totalTokens | Int? | | 总 Token 消耗 |
| totalCost | Float? | | 总成本（元） |
| startedAt | DateTime | DEFAULT now() | 开始时间 |
| completedAt | DateTime? | | 完成时间 |
| duration | Int? | | 耗时（毫秒） |

## 索引设计

```prisma
// User
@@index([email])                       // 登录查询

// Couple
@@index([slug])                        // 公开页面路由查询

// CoupleUser
@@unique([userId, coupleId])           // 防重复加入
@@index([userId])                      // 查用户所属空间
@@index([coupleId])                    // 查空间成员

// Album
@@index([coupleId, sortOrder])         // 空间下相册列表

// Photo
@@index([albumId, sortOrder])          // 相册内照片列表
@@index([albumId, takenAt])            // 按拍摄时间排序
@@index([status])                      // 按状态筛选（处理队列）

// Milestone
@@index([coupleId, date])              // 时间轴查询

// PipelineRun
@@index([photoId])                     // 查照片的处理记录
@@index([coupleId, startedAt])         // 空间下的执行历史
```

## Prisma Schema 核心片段

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

enum Role {
  OWNER
  PARTNER
}

enum PhotoStatus {
  UPLOADING
  PROCESSING
  READY
  FAILED
}

enum PipelineStatus {
  RUNNING
  COMPLETED
  FAILED
}

model User {
  id           String       @id @default(cuid())
  email        String       @unique
  passwordHash String
  name         String?
  avatar       String?
  couples      CoupleUser[]
  createdAt    DateTime     @default(now())
  updatedAt    DateTime     @updatedAt
}

model Couple {
  id            String         @id @default(cuid())
  slug          String         @unique
  name          String
  startDate     DateTime?
  coverPhotoUrl String?
  bio           String?
  theme         String         @default("default")
  isPublic      Boolean        @default(false)
  plan          String         @default("free")
  members       CoupleUser[]
  albums        Album[]
  milestones    Milestone[]
  pipelineRuns  PipelineRun[]
  createdAt     DateTime       @default(now())
  updatedAt     DateTime       @updatedAt
}

model CoupleUser {
  id       String   @id @default(cuid())
  user     User     @relation(fields: [userId], references: [id])
  userId   String
  couple   Couple   @relation(fields: [coupleId], references: [id])
  coupleId String
  role     Role
  nickname String?
  joinedAt DateTime @default(now())

  @@unique([userId, coupleId])
  @@index([userId])
  @@index([coupleId])
}

model Album {
  id            String   @id @default(cuid())
  couple        Couple   @relation(fields: [coupleId], references: [id])
  coupleId      String
  title         String
  description   String?
  coverPhotoUrl String?
  sortOrder     Int      @default(0)
  photos        Photo[]
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@index([coupleId, sortOrder])
}

model Photo {
  id              String      @id @default(cuid())
  album           Album       @relation(fields: [albumId], references: [id])
  albumId         String
  originalUrl     String
  thumbnailUrl    String?
  displayUrl      String?
  fileName        String
  fileSize        Int
  width           Int?
  height          Int?
  sortOrder       Int         @default(0)
  // EXIF
  takenAt         DateTime?
  latitude        Float?
  longitude       Float?
  locationName    String?
  cameraMake      String?
  cameraModel     String?
  focalLength     String?
  aperture        String?
  shutterSpeed    String?
  iso             Int?
  // AI
  aiCaption       String?
  userCaption     String?
  aiScene         String?
  aiMood          String?
  aiComposition   String?
  aiColorTone     String?
  aiLayout        String      @default("side-by-side")
  aiKeywords      String[]
  // Status
  status          PhotoStatus @default(UPLOADING)
  processingError String?
  milestones      Milestone[]
  pipelineRuns    PipelineRun[]
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt

  @@index([albumId, sortOrder])
  @@index([albumId, takenAt])
  @@index([status])
}

model Milestone {
  id              String   @id @default(cuid())
  couple          Couple   @relation(fields: [coupleId], references: [id])
  coupleId        String
  title           String
  description     String?
  date            DateTime
  locationName    String?
  photo           Photo?   @relation(fields: [photoId], references: [id])
  photoId         String?
  isAutoGenerated Boolean  @default(false)
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@index([coupleId, date])
}

model PipelineRun {
  id          String         @id @default(cuid())
  couple      Couple         @relation(fields: [coupleId], references: [id])
  coupleId    String
  photo       Photo          @relation(fields: [photoId], references: [id])
  photoId     String
  status      PipelineStatus @default(RUNNING)
  dag         Json
  nodeResults Json?
  totalTokens Int?
  totalCost   Float?
  startedAt   DateTime       @default(now())
  completedAt DateTime?
  duration    Int?

  @@index([photoId])
  @@index([coupleId, startedAt])
}
```

## 数据迁移策略

使用 Prisma Migrate 管理数据库迁移：

- 开发环境：`prisma migrate dev` — 自动生成并应用迁移
- 生产环境：`prisma migrate deploy` — 仅应用已有迁移
- 迁移文件纳入 Git 版本控制
- 破坏性变更（删列/改类型）需手写迁移 SQL，不依赖自动生成
