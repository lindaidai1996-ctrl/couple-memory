# 产品需求文档

> 本文档定义用户角色、MVP 功能清单、核心用户流程和非功能需求。
>
> 相关文档：[总览](./README.md) · [技术架构](./architecture.md) · [API 设计](./api-design.md)

## 用户角色

| 角色 | 身份 | 权限范围 |
|------|------|---------|
| OWNER | 创建空间的用户 | 全部操作：上传、编辑、删除、管理空间、邀请 PARTNER |
| PARTNER | 被邀请加入空间的伴侣 | 上传、编辑照片和文案，不能删除空间 |
| 访客 | 通过公开链接访问的人 | 只能浏览公开页面，无编辑权限 |

一个用户可以是多个空间的成员（通过 CoupleUser 关联表），但 MVP 阶段每人限一个空间。

## MVP 功能清单

### F1: 用户注册与登录

用户通过邮箱 + 密码注册账号。NextAuth v5 Credentials Provider 实现，预留 OAuth Provider 扩展能力。

注册时自动创建一个"空间（Couple）"，用户成为 OWNER。可生成邀请链接让伴侣加入成为 PARTNER。

### F2: 多租户空间管理

每对情侣一个独立空间，通过唯一 `slug` 标识。公开页面访问地址为 `/s/{slug}`。

空间包含：名称、开始日期（在一起的日子）、主题配置、是否公开。OWNER 可编辑空间信息和公开状态。

### F3: 照片上传与 EXIF 提取

用户在后台上传照片，前端使用 browser-image-compression 预压缩（5MB 以内，保留 EXIF），然后直传阿里云 OSS（不经服务端）。

上传完成后，后端异步处理：
- Sharp 生成 3 个尺寸（缩略图 300px、展示图 1200px、原图）
- exifr 提取 EXIF 信息（拍摄时间、GPS 坐标、相机型号、光圈快门等）
- GPS 坐标通过高德 API 逆地理编码为中文地名

详见 [图片处理管道](./image-pipeline.md)。

### F4: AI 自动生成文案

基于照片内容分析结果（场景、情绪、构图）+ 地点 + 时间，由 CaptionWriter Agent（DeepSeek）生成浪漫文案。

生成的文案存储为 `aiCaption`，用户可自行编写 `userCaption` 覆盖。展示时优先使用 userCaption。

支持多种文案风格：浪漫絮语、诗意短句、日记体、摄影手记。

### F5: AI 智能排版

LayoutAdvisor Agent 根据照片的宽高比、构图分析、色调，从 5 种预设模板中选择最佳排版：

| 模板 | 适用场景 |
|------|---------|
| cinema-wide | 宽幅风景照，电影感横构图 |
| side-by-side | 照片 + 文字并排，博客式叙事 |
| portrait-hero | 竖构图人像，大图冲击力 |
| grid-square | 方形构图或多图组合 |
| story-card | 卡片式，适合社交分享 |

用户可在后台手动切换排版模板。

### F6: 时间轴自动生成

TimelineBuilder Agent 根据照片的拍摄时间和地点，自动识别"里程碑"时刻（如首次到某地、特殊日期）。

时间轴展示在公开页面的"我们的足迹"板块，支持按时间线性浏览。里程碑分为 AI 自动生成（isAutoGenerated=true）和用户手动创建两种。

### F7: 公开页面展示

访客通过 `/s/{slug}` 访问情侣的公开页面。页面包含：

- **首屏 Hero**：封面照片 + 情侣名 + 在一起天数
- **照片集**：按相册或时间排列，每张照片配文案，使用 AI 推荐的排版模板
- **时间轴**："我们的足迹"，里程碑事件 + 时间 + 地点
- **关于我们**：自定义的简介内容

页面需做到：摄影师级美学、响应式（桌面 + 移动端）、图片懒加载。

### F8: 后台管理面板

OWNER 和 PARTNER 登录后进入后台，功能包括：

- 空间设置：名称、slug、开始日期、公开状态
- 相册管理：创建/编辑/删除相册，调整排序
- 照片管理：上传、查看处理状态、编辑文案、切换排版模板、删除
- 里程碑管理：查看/编辑/删除自动生成的里程碑，手动创建里程碑
- AI 处理状态：查看 Pipeline 执行记录（调试用，可选展示）

## 核心用户流程

### 流程 1：创建者上传照片（核心路径）

```
注册/登录
  → 创建空间（填写名称、slug、开始日期）
  → 进入后台，上传照片
  → 前端预压缩 → OSS 直传 → 后端异步处理
  → AI Pipeline 执行（分析 → 文案 + 排版 + 时间轴）
  → 处理完成，照片状态变为 READY
  → 预览公开页面效果
  → 可选：编辑文案、切换排版
  → 设为公开，分享链接
```

### 流程 2：访客浏览公开页

```
收到链接 /s/{slug}
  → 访问公开页面（无需登录）
  → 浏览首屏 Hero → 照片集 → 时间轴
  → （MVP 后）留言/祝福
```

### 流程 3：邀请伴侣

```
OWNER 在后台生成邀请链接
  → 伴侣点击链接 → 注册/登录
  → 自动加入空间成为 PARTNER
  → 可上传照片、编辑文案
```

## 扩展功能（MVP 后）

以下功能已规划但不在 MVP 范围内：

- 婚礼邀请函场景（RSVP 确认出席）
- 纪念日自动提醒 + AI 生成"周年回顾"
- 访客留言/祝福墙
- 一键生成分享海报（适配朋友圈/小红书）
- AI 故事生成（照片串联叙事）
- AI 风格迁移/滤镜（电影感/胶片感/日系）
- AI 语音旁白 + 背景音乐（有声相册）
- AI 对话记忆检索（"我们第一次去海边是什么时候？"）
- 微信/手机号登录
- 付费计划与升级

## 非功能需求

| 维度 | 要求 |
|------|-----|
| 首屏加载 | < 3 秒（国内网络环境） |
| 图片加载 | 懒加载 + 渐进式加载（先缩略图后高清） |
| 移动端适配 | 响应式设计，核心功能在移动端可用 |
| SEO | 公开页面支持 SSR/SSG，Open Graph 元标签 |
| 安全 | 密码 bcrypt 哈希、OSS 临时凭证（STS）、CSRF 防护 |
| 可用性 | 单机部署，目标 99% 可用率 |
| 数据备份 | PostgreSQL 定期备份（pg_dump） |
