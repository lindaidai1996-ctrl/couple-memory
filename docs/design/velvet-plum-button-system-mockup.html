Design Decisions

按钮体系定义

这一版是为了确认公共组件边界和视觉语言，后续再沉淀成真实 tokens 与组件 API。

Recommended
Variant mapproposal
  • brand: 默认主操作，沿用现有 `--vp-active-gradient-*` 的酒红材质。
  • success: 成功或完成动作，色相偏鼠尾草绿，但保留同样的内高光与阴影结构。
  • warning: 提醒、催办、谨慎继续，色相偏温琥珀，不做互联网橙黄高饱和。
  • danger: 删除、清空、不可逆操作，色相偏酒红玫瑰，和品牌色同宗但危险度更高。
  • secondary / subtle / ghost: 不承担强语义，只作为表层优先级差异。
API shapedraft
  • variant: `brand | success | warning | danger | secondary | subtle | ghost`
  • size: `sm | md | lg`，默认 `md`
  • icon: 支持 `leadingIcon`、`trailingIcon` 和 `icon-only`
  • state: `loading`、`disabled`、`aria-busy`、`data-destructive`
Interaction rulesmust keep
  • Hover: 只允许轻微上浮或对比加深，不做夸张缩放。
  • Focus: 统一使用 plum / rose ring，禁止浏览器默认蓝色。
  • Disabled: 保留形态轮廓，但取消浮起和强阴影。
  • Loading: 文案和 spinner 一起存在，不只留一个旋转图标。
Implementation notenext step
如果你确认方向没问题，下一步我会把这套视觉映射成真实公共组件，包括 token 命名、variant API、暗亮主题变量和状态类。
如果你想调整，我建议优先决定两件事：`warning 是否太暖`，以及 `danger 和 brand 是否还需要更大区分`。
