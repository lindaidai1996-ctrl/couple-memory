# AI 纪念站生成器 V1 设计

> 适用范围：基于已有情侣空间、相册、章节、照片与基础元数据，自动生成一个单页式、摄影师纪实风的专属纪念站。

## 背景与判断

当前产品已经具备相册、章节、时间线、公开页、AI 文案和回顾能力，主问题不再是“能不能展示照片”，而是“能不能把共同记忆整理成一个值得分享的成品”。

如果继续把产品表述为“AI 相册站”，收费点会偏弱，因为用户感知到的是工具能力；如果把产品升级为“AI 自动策展的专属纪念网站生成器”，收费点会更清晰，因为用户购买的是审美结果、叙事结果和可分享的成品。

因此，V1 不做通用 AI 建站器，而做一个高审美成品生成器。

## 产品定义

一句话定位：

`AI 帮两个人把已有照片和相册，自动策展成一个像摄影师交付件的专属纪念网站。`

V1 的核心承诺：

1. 不要求用户自己搭站。
2. 不要求用户逐张选图。
3. 输出的是一个成品页，而不是一个编辑器。

## 设计目标

1. 基于已有内容自动生成一个单页式、高审美、可分享的纪念站。
2. 当照片很多、相册很多时，首页仍然保持节奏感和成品感。
3. 让 AI 负责重策展，用户只做轻确认和少量微调。
4. 为后续付费转化提供清晰商品形态。

## 非目标

1. 不做通用网站 Builder。
2. 不做多页面站点作为 V1 主形态。
3. 不追求首页全量承载所有照片和所有相册。
4. 不提供自由拖拽布局、自由加模块、自由改主题等重编辑能力。
5. 不把首页做成完整归档页或无限照片流。

## 核心方案

### 1. 产物形态：单页式纪实特刊流

V1 固定生成一个单页式纪念站，阅读感应接近“线上纪念特刊”，而不是“相册目录”。

页面分为四个稳定层级：

1. 封面段：主图、总标题、副标题、关系时间。
2. 关系引子段：极短文字，交代这段纪念站的情绪和时间跨度。
3. 阶段篇章段：按章节或阶段组织的主体内容。
4. 收尾段：尾图、总结句、完整内容入口。

### 2. 内容策略：精选展示，不全量平铺

首页不承担“完整展示全部内容”的任务，而承担“用最少内容讲出最有感觉的故事”的任务。

固定约束：

1. 首页只展示 `4` 到 `7` 个阶段。
2. 每个阶段展示 `3` 到 `7` 张图。
3. 总图数尽量控制在 `25` 到 `45` 张。
4. 未进入首页的内容不删除，而是进入完整章节或完整相册入口。

### 3. 精选逻辑：按章节或阶段精选

V1 的策展单位不是单张照片，也不是时间线碎片，而是章节或阶段。

推荐流程：

1. 从候选章节中选出最适合首页讲述的 `4` 到 `7` 个阶段。
2. 为每个阶段生成一个适合阅读的篇章标题。
3. 从每个阶段内部选出代表图，组合成一段完整篇章。

这样可以避免首页变成“按相册名机械罗列”的内容流。

### 4. 选片结构：平衡型策展

V1 的选片策略不是只挑“最清晰”或“最高分”的图，而是按平衡型结构选图：

1. 一张主图：代表该阶段整体气质。
2. 若干人物关系图：体现两个人的互动。
3. 若干场景氛围图：提供地点、旅行或生活环境感。
4. 一到两张细节图：补足情绪与节奏。

目标不是挑“最好看”的几张，而是拼出“像摄影师选片后的成组结果”。

### 5. 页面节奏：篇章式排版，而非重复卡片

阶段篇章需要维持统一审美，但不能全部长成同一张卡片。V1 可在固定骨架内轮换少量版式：

1. 主图在左，组图在右。
2. 主图在上，文字和小图在下。
3. 双栏纪实排版。

章节之间允许插入轻量过渡信息，如时间变化、地点变化或关系阶段提示，避免硬切和流水账感。

## 用户流程

### 1. 选择素材范围

用户只做轻量范围确认：

1. 选择一个相册，或选择若干章节。
2. 选择一个整体风格，V1 默认主推“摄影师纪实风”。

此处不允许逐张选图，不允许复杂配置。

### 2. AI 自动生成

系统自动完成：

1. 选阶段。
2. 选图。
3. 生成总标题与关系引子。
4. 生成阶段标题和阶段短说明。
5. 决定篇章顺序和篇章版式。
6. 生成完整单页预览。

### 3. 用户审核

V1 只提供三类微调：

1. 换一版选图。
2. 替换某一张图。
3. 修改文案。

V1 不提供自由拖拽布局或自由重组页面结构。

### 4. 确认发布

用户确认后发布，得到正式可分享链接。

## AI 与用户分工

AI 负责：

1. 策展。
2. 编排。
3. 文案起草。
4. 风格统一。
5. 页面生成。

用户负责：

1. 选择范围。
2. 审核结果。
3. 做少量替换和确认。

产品原则是：让“再生成一版”比“手工改很多”更容易。

## 数据与输入边界

V1 生成器优先使用当前系统已经稳定存在的内容：

1. 相册标题与相册说明。
2. 章节标题与章节摘要。
3. 照片时间、地点、基础分析结果。
4. 用户已有文案与 AI 草稿。

V1 不依赖新增大规模人工标注。已有结构越完整，生成结果越稳。

## 收费模型

V1 以按成品收费为主，而不是以订阅为主。

建议模型：

1. 免费预览：允许用户先生成一个预览结果。
2. 单次生成付费：用户为正式生成和正式发布付费。
3. 增值项付费：后续可扩展为额外风格版本、下载型成品、专题页和专属域名。

收费对象应明确为：

`AI 为你们自动生成一份摄影师风格的专属纪念网站。`

## 成功标准

1. 用户能在少量操作后拿到一个看起来像成品的专属页面。
2. 首页在“照片很多、章节很多”的情况下仍然保持清晰节奏。
3. 用户愿意分享该页面，而不是只把它当后台预览。
4. 用户对“按成品收费”有清晰感知，不需要先理解复杂会员权益。

## 风险与控制

### 风险 1：结果像模板站，不像专属作品

控制方式：

1. 固定纪实特刊骨架。
2. 把差异放在选片、文案和篇章节奏，而不是只换配色和字体。

### 风险 2：首页内容过多，失去成品感

控制方式：

1. 严格限制阶段数和总图数。
2. 把剩余内容导向完整章节或完整相册。

### 风险 3：为了缓解用户不信任而把编辑能力做重

控制方式：

1. 只开放少量替换入口。
2. 强化“再生成一版”而不是强化自由编辑。

## 明确不做什么

V1 不做：

1. 自由拖拽网站编辑器。
2. 多页面建站流程。
3. 首页全量照片展示。
4. 高复杂度主题自定义。
5. 把通用建站能力作为卖点。

## 后续演进方向

如果 V1 验证通过，后续可以继续扩展：

1. 多风格生成：恋爱杂志风、电影回忆风。
2. 专题站：一周年、旅行、1000 天等主题站。
3. 多页面小站：在单页成品稳定后再扩首页之外的页面。
4. 衍生成品：长图、PDF 纪念册、分享海报、视频回顾。

V1 的边界必须保持克制，先证明“单页纪实特刊流生成器”本身能打动用户并支撑收费。
